# Reflection – Kanban Board Implementation and Template Selection

During this assignment, I explored GitHub Project templates and implemented a Kanban board to support Agile development for the Aura Reels Movie Rental System. While the process was structured, several challenges were encountered when selecting the most suitable template and customising the board to fit the system’s needs.

One of the main challenges was choosing the appropriate template. GitHub provides multiple templates such as Basic Kanban, Automated Kanban, Bug Triage, and Team Planning. Initially, it was difficult to determine which template would best support Agile development. Some templates focused more on issue tracking rather than full project planning. After evaluating their features, I selected the Team Planning template because it supports backlog management, task prioritisation, and structured workflow stages, which align well with Agile practices.

Another challenge was customising the Kanban board. Although the default template provided basic columns, it did not fully represent the development lifecycle of the system. I had to think carefully about additional workflow stages and decided to include columns such as "Testing" and "Blocked". Adding these columns improved visibility and allowed better tracking of tasks that require quality assurance or are delayed due to dependencies.

Linking issues from previous assignments was also a key challenge. It required ensuring that all user stories from Assignment 6 were correctly represented as GitHub Issues and then linked to the Kanban board. This process highlighted the importance of consistency and traceability across assignments, as each stage builds on the previous one.

Working independently was another challenge. In real-world Agile environments, teams collaborate and distribute tasks among members. In this assignment, I had to manage all roles myself, including planning, development, and task tracking. This required strong time management and self-discipline, as well as careful decision-making when assigning tasks and managing workflow.

When comparing GitHub Projects to other tools such as Trello and Jira, it became clear that GitHub is more integrated with development workflows. While Trello provides a more visual and user-friendly interface, and Jira offers advanced project management features, GitHub Projects is highly effective for developers because it directly integrates with repositories, issues, and version control. This makes it easier to manage tasks and track progress in one place.

Overall, this assignment improved my understanding of Agile methodologies and Kanban boards. It demonstrated the importance of workflow visualisation, task management, and adaptability in software development. It also highlighted how project management tools can be customised to meet specific system requirements, ensuring efficient and organised development.
