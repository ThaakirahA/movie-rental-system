# Reflection – Use Case Modeling and Test Case Development

This assignment focused on translating system requirements into use case models and test cases for the Aura Reels Movie Rental System. One of the main challenges encountered was ensuring that all use cases were clearly aligned with the functional requirements defined in Assignment 4.

Initially, it was difficult to decide how detailed each use case specification should be. Some use cases, such as “Rent Movie,” required multiple steps and validation checks, while others like “Browse Movies” were simpler. Balancing clarity and completeness without making the specifications overly complex was a key challenge.

Another challenge was identifying appropriate alternative flows. Real-world systems often have many possible exceptions, such as invalid inputs or unavailable movies. Ensuring that these scenarios were included helped improve the robustness of the system design.

Creating the use case diagram also required careful consideration of stakeholders. The system includes multiple actors such as customers, administrators, staff, developers, and business owners. Mapping their interactions accurately ensured that the system design reflected real-world usage and stakeholder concerns identified in Assignment 4.

Test case development presented another important challenge. Each test case had to clearly map to a functional requirement, ensuring traceability. It was important to define precise expected results so that system behavior could be validated effectively. Additionally, non-functional testing required thinking beyond basic functionality to consider performance and security aspects, such as system response time and protection against unauthorized access.

One key lesson learned from this assignment is the importance of traceability between requirements, use cases, and test cases. Each stage builds on the previous one, and maintaining consistency ensures that the system design remains coherent and complete.

Overall, this assignment improved my understanding of how to model user interactions and validate system behavior. It demonstrated how use cases and test cases play a critical role in ensuring that software systems meet stakeholder needs and function correctly under various conditions. 

Additionally, creating a traceability matrix highlighted the importance of linking requirements, use cases, and test cases. This ensured that no requirement was left untested and improved the overall quality and reliability of the system design.
