# Reflection – Use Case Modeling and Test Case Development

This assignment focused on translating the system requirements from Assignment 4 into comprehensive use case models and detailed test cases for the **Aura Reels Movie Rental System**. The objective was to ensure that every functional requirement could be clearly traced through use cases and validated via test scenarios, while also addressing non-functional concerns such as performance, security, and scalability.

One of the first challenges encountered was **defining the appropriate actors and their roles**. Initially, it was tempting to consolidate administrators, staff, and system administrators into a single actor, as their responsibilities overlap in certain areas. However, a careful analysis of the stakeholder concerns highlighted in Assignment 4 revealed important distinctions. Administrators primarily manage the movie catalog and user accounts, staff assist with operational tasks, and system administrators focus on security, server maintenance, and uptime. Separating these actors allowed for more accurate mapping of use cases to real-world stakeholder interactions, ensuring that all concerns were addressed.

Another significant challenge was **determining the level of detail for each use case specification**. Some use cases, such as “Rent Movie,” required multiple steps and validation checks, including login verification, stock availability, and notification of successful rentals. Other use cases, like “Browse Movies,” were simpler but still required clear preconditions, postconditions, and alternative flows to handle scenarios like no available movies or search errors. Striking a balance between clarity and completeness without making specifications overly complex was a key focus.

**Alternative flows** presented an additional challenge. Real-world applications encounter numerous exceptions, such as invalid inputs, unavailable movies, or failed system actions. Including these scenarios in the use case specifications helped to improve the robustness of the system design and ensured that the system could handle errors gracefully without compromising the user experience.

**Test case development** required careful alignment with both functional and non-functional requirements. Each test case was mapped directly to a requirement from Assignment 4, ensuring full traceability. It was crucial to define precise steps and expected results so that system behavior could be validated accurately. For non-functional requirements, scenarios were designed to test system performance under **1,000 concurrent users**, verify response times of ≤2 seconds, and validate security measures such as authentication, password encryption, and protection against unauthorized access or brute-force attacks.

An important lesson learned from this assignment is the **critical role of traceability**. Linking requirements, use cases, and test cases ensures that all stakeholder needs are addressed and that no system functionality is left untested. This process also highlighted gaps in the original requirements and informed minor adjustments to improve usability and system robustness.

Overall, this assignment significantly enhanced my understanding of **modeling user interactions and validating system behavior**. Creating the use case diagram provided a visual representation of system interactions, while detailed specifications and test cases ensured comprehensive coverage of system functionality. The exercise demonstrated how structured modeling and testing can improve system reliability, performance, and stakeholder satisfaction, laying a strong foundation for subsequent development and testing phases.

In conclusion, Assignment 5 emphasized the importance of detailed planning, stakeholder alignment, and rigorous testing. By translating requirements into use cases and test cases, the Aura Reels Movie Rental System design is now more robust, traceable, and aligned with real-world usage, ensuring both functional and non-functional expectations are met.
