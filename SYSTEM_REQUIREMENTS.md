# System Requirements – Aura Reels Movie Rental System

## Functional Requirements

1. The system shall allow users to register an account.
2. The system shall allow users to log into the system securely.
3. The system shall allow users to browse a catalogue of available movies.
4. The system shall allow users to search for movies by title.
5. The system shall display detailed movie information including title, description, genre, and rating.
6. The system shall allow users to add movies to a rental cart.
7. The system shall allow users to remove movies from the cart.
8. The system shall allow users to complete a rental checkout process.
9. The system shall allow administrators to add new movies to the system.
10. The system shall allow administrators to edit existing movie information.
11. The system shall allow administrators to delete movies from the system.
12. The system shall allow users to view movie trailers before renting.

### Acceptance Criteria

* Search results must appear within **2 seconds**.
* Movie availability must update **immediately after a rental**.
* Users must receive a confirmation notification after completing a rental.

---

## Non-Functional Requirements

### Usability

The system shall provide a user-friendly interface that allows users to browse and rent movies easily.

### Deployability

The system shall be deployable on web servers supporting Node.js.

### Maintainability

The system shall follow a modular architecture separating frontend, backend, and database layers.

### Scalability

The system shall support at least **1000 concurrent users**.

### Security

User authentication and sensitive data shall be protected using encryption.

### Performance

The system shall load search results within **2 seconds**.

### Reliability

The system shall maintain **99% uptime**.

### Availability

The system shall be accessible via modern web browsers at all times.
