# Reflection – Agile Planning and Personal Challenges

During the development of the Aura Reels Movie Rental System, applying Agile methodology introduced several challenges, particularly in prioritisation, effort estimation, and aligning tasks with system requirements.

One of the main challenges was breaking down system requirements into user stories. Although the requirements from Assignment 4 were clearly defined, converting them into user stories required a shift in thinking from system-focused descriptions to user-centered needs. It was sometimes difficult to ensure that each story clearly expressed value from the user’s perspective while still remaining technically accurate and aligned with the system design.

Another challenge was prioritising user stories using the MoSCoW method. Many of the system features seemed equally important, which made it difficult to decide what should be classified as “Must-have” versus “Should-have” or “Could-have.” This required careful consideration of what features were essential for delivering a Minimum Viable Product (MVP). Core features such as registration, login, browsing, searching, and renting movies were prioritised as Must-have because they form the primary functionality of the system.

Effort estimation was also challenging. Assigning story points required estimating the complexity of tasks without actually implementing them. This introduced uncertainty, as some tasks that appear simple at first could become more complex during development. Balancing realistic estimates while avoiding overestimation required thoughtful judgement and understanding of the system architecture.

Another significant challenge was working independently. In a real Agile environment, different team members would collaborate, share responsibilities, and support each other. However, in this assignment, I had to take on multiple roles, including product owner, developer, and tester. This created internal resistance at times, as maintaining motivation and managing workload alone required strong self-discipline and time management.

Sprint planning also required careful decision-making. Selecting the appropriate number of user stories for a 2-week sprint involved balancing ambition with realism. Including too many stories could lead to incomplete work, while including too few would not fully demonstrate the system’s capabilities. Focusing on core features ensured that the sprint remained achievable while still delivering meaningful progress.

Despite these challenges, applying Agile methodology provided a structured approach to organising development tasks and prioritising work effectively. It improved clarity by linking user stories to system requirements and ensured that each feature had a clear purpose and measurable outcome.

Overall, this assignment highlighted the importance of prioritisation, estimation, and adaptability in Agile development. It also emphasised the importance of self-discipline when working independently and the need to balance technical requirements with user needs to deliver a functional and user-friendly system.
