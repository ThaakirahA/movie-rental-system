# 🎬 Domain Model  
## Aura Reels Movie Rental System

This domain model defines the key entities, their attributes, methods, relationships, and business rules within the system.

---

## 📊 Domain Entities

| Entity | Attributes | Methods | Relationships |
|--------|-----------|---------|--------------|
| User | userId, name, email, password | register(), login(), logout() | Creates Rental, owns RentalCart |
| Movie | movieId, title, genre, status | viewDetails(), updateAvailability() | Associated with Rental |
| Rental | rentalId, rentalDate, status | createRental(), cancelRental() | Linked to User and Movie |
| Payment | paymentId, amount, status | processPayment(), validatePayment() | Linked to Rental |
| RentalCart | cartId, items | addMovie(), removeMovie() | Owned by User |
| Admin | adminId, name | manageMovies(), generateReport() | Manages Movie, generates Report |
| Report | reportId, date | generateReport(), exportReport() | Generated by Admin |

---

## 🔗 Relationships

- A **User** can create multiple **Rentals**
- A **Rental** is linked to one **Movie**
- A **Rental** requires one **Payment**
- A **User** owns one **RentalCart**
- A **RentalCart** contains multiple **Movies**
- An **Admin** manages **Movies**
- An **Admin** generates **Reports**

---

## 📌 Business Rules

- A user must be logged in to rent a movie
- A movie must be available before it can be rented
- A payment must be successful before confirming a rental
- A user can have multiple rentals but must complete payment for each
- A cart must contain at least one movie before checkout
- Admins are responsible for maintaining movie availability
